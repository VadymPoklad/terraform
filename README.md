# Інфраструктура для Jira Clone (AWS & Terraform)

Цей репозиторій містить конфігурацію "Інфраструктура як Код" (Infrastructure as Code) на базі Terraform для автоматизованого розгортання повноцінного вебдодатку (Jira Clone на Next.js) у хмарі AWS.

## Архітектура

Система розгортається у високодоступному (Highly Available) та захищеному середовищі:

- **VPC (Virtual Private Cloud)**: Спеціально виділена мережа з публічними, приватними та ізольованими підмережами бази даних у двох зонах доступності (`us-east-1a`, `us-east-1b`).
- **БД (Database)**: AWS RDS PostgreSQL (t3.micro) розміщена у повністю приватних підмережах.
- **Обчислювальні ресурси (Compute)**: EC2 інстанси (t3.small) розміщені у приватних підмережах з доступом до інтернету через NAT Gateways.
- **Автомасштабування (Auto Scaling Group)**: Група автоматично підтримує необхідну кількість інстансів (2) та перевіряє їх стан.
- **Балансування (ALB)**: Application Load Balancer розподіляє вхідний трафік між серверами.
- **Управління секретами**: Використовується AWS Secrets Manager для зберігання змінних оточення (Ключі Clerk, Upstash Redis, паролі БД). Сервери самостійно стягують їх при старті завдяки налаштованій IAM Ролі.

## Передумови

Перед початком переконайтесь, що у вас встановлено:
1. Terraform
2. AWS CLI (авторизований через `aws configure`)
3. Створено необхідні секрети в AWS Secrets Manager:
   - `prod/app` (з ключами `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY`, `CLERK_SECRET_KEY` тощо)
4. У вас є готовий образ AMI (Amazon Machine Image), який містить зібраний додаток.

## Як розгорнути

1. **Ініціалізація Terraform** (Завантаження модулів та провайдерів):
   ```bash
   terraform init
   ```

2. **Оновлення AMI** (за потреби):
   У файлі `main.tf` оновіть змінну `ami_id` на ваш останній "золотий" образ:
   ```hcl
   ami_id = "ami-ВАШ_ID"
   ```

3. **Перевірка плану**:
   Перевірте, які ресурси будуть створені:
   ```bash
   terraform plan
   ```

4. **Застосування змін**:
   Розгорніть інфраструктуру:
   ```bash
   terraform apply
   ```
   *Потрібно буде підтвердити дію, ввівши `yes`.*

## Як отримати доступ

Після успішного виконання `terraform apply`, ви побачите вихідні дані (Outputs) у консолі. 
Щоб дізнатись URL вашого сайту, виконайте команду:

```bash
terraform output alb_dns_name
```
Скопіюйте це посилання у браузер (наприклад: `jira-clone-alb-123456.us-east-1.elb.amazonaws.com`).

## Як видалити ресурси

Щоб уникнути зайвих витрат в AWS, коли проєкт більше не потрібен (наприклад, після завершення тестування), обов'язково видаліть усі ресурси:

```bash
terraform destroy
```
*Примітка: Якщо виникає помилка "DependencyViolation" для Security Groups, просто зачекайте 1-2 хвилини і запустіть `terraform destroy` ще раз. Це нормальна поведінка AWS при видаленні баз даних і мережевих інтерфейсів.*

## Структура коду

* `main.tf` - Головний файл, який збирає всі модулі.
* `outputs.tf` - Вихідні змінні проєкту.
* `modules/vpc/` - Створення мережі, підмереж, шлюзів та таблиць маршрутизації.
* `modules/ec2/` - Налаштування серверів, IAM ролей, ALB, та ASG.
* `modules/db/` - Налаштування захищеної бази даних RDS.
